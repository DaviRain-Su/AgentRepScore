## Codebase Patterns
- **Script entry guard**: All new CLI scripts in `scripts/` should use `isMainModule()` (same pattern as `src/cli.ts` and `src/server.ts`) to prevent accidental execution during test imports.
- **Keeper submission utilities**: Use `src/skill/keeper-utils.ts` for all on-chain `submit*` calls. It provides exponential-backoff retry (`submitWithRetry`), evidence-hash deduplication (`isAlreadySubmitted`), and local state persistence (`loadKeeperState` / `saveKeeperState`).
- **Keeper state file**: The local deduplication state file path defaults to `.keeper-state.json` and can be overridden with the `KEEPER_STATE_PATH` env var.
- **Testnet RPC sensitivity**: `test/hardhat/integration.test.ts` runs against live X Layer testnet and will timeout/hang if the RPC is slow or unreachable. This can block `pnpm test:all` even when local/unit changes are correct.
- **Pool address configuration**: Uniswap V3 pool addresses are not hardcoded; they must be supplied via `UNISWAP_POOLS` env var as comma-separated addresses.
- **Struct ↔ ABI sync**: When adding a field to a Solidity struct consumed by the indexer/keeper, update **both** the TypeScript `SwapSummary` interface in `src/skill/eip712.ts` and the inline ABI tuple in `scripts/indexer-uniswap.ts` (or equivalent). `viem` type inference will reject mismatched tuples at compile time.
- **Solidity reserved words**: `reference` is a reserved keyword in Solidity 0.8.20 and cannot be used as a local variable or parameter name (choose `ref` instead).
